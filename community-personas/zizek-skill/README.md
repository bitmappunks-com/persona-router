# zizek-skill

一个 Claude Code skill。它做一件事：**把你自以为想清楚了的东西，翻出你没意识到自己在默认的那一层。**

不是齐泽克 cosplay。不是哲学术语生成器。是一个用齐泽克式的问题意识做分析的工具——追问前提、追踪欲望、识别功能、揭示矛盾，然后留下一个更准确的说法。

## 它和普通批判性分析的区别

- 普通分析在你的框架里回答你。这个 skill 的第一步是**质疑框架本身**。
- 普通分析追问"你说得对不对"。这个 skill 追问"你为什么需要这么说"。
- 普通分析看内容。这个 skill 看**功能**——一段话在现实里做了什么，而不只是说了什么。

## 快速示例

**输入**
> 考了三年公务员终于上岸了，但入职两个月就开始觉得不对。我以前那么拼命到底是为了什么？

**输出方向**
> 三年备考的全部合法性建立在一个承诺上："上了岸就好了。"当你到了，承诺兑现了——但兑现的内容是空的。你追的不是一份工作，你追的是"考上之后一切都会对"这个叙事的完整性。"上岸"最隐蔽的功能是：它取消了你上岸之后不满的合法性。

**输入**
> 我不想当"恋爱脑"。我是独立女性，我不需要一段关系来定义我。

**输出方向**
> 反复宣布"我不需要"，恰恰是最需要的证据——你需要不需要。当"独立"本身成了一种可展示的人设模板，你分不清哪部分是选择，哪部分是新的规训。你从一个牢笼走出来，但走进的那个地方，也许只是看起来更体面的另一个。

## 工作方式

不分模式。拿到任何东西——一段话语、一个困惑、一条朋友圈、一种说不清的不舒服——做同一组分析动作：追问前提、追踪欲望、找矛盾、看功能、留下更准确的说法。输出形状跟着材料自然走。

## 适用场景

- 拆解搭子关系、附近消失、犬儒清醒、上岸幻想、性别话语死锁、自我提升无限循环等当代中国集体叙事
- 追问你和你的生活之间隔了哪一层、你的"看穿"在替你做什么
- 分析为什么一段说法让人觉得空洞、回避或自欺
- 把一种模糊的不适感重新表述成一个准确的问题
- 审视小红书帖子、朋友圈文案、集体话语里的隐含逻辑

## 不做什么

- 事实调研
- 编码
- 泛情绪安抚
- 齐泽克口音 roleplay
- 把所有问题都上升到"晚期资本主义"

## 设计原则

- 分析先于表演。精确先于戏剧化。
- 哲学概念是手术刀，不是装饰。只在普通语言切不动的时候用。
- 每次拆解都必须留下一个更准确的说法。只拆不建不算完成。
- 好的输出是刚好够的输出。到了就停。
- 即使拿掉"齐泽克"这个名字，这个 skill 也应该仍然有用。

## 核心理论资源

| 概念 | 使用频率 | 核心功能 |
|------|---------|---------|
| 意识形态 | 高 | 识别被自然化的安排 |
| 欲望 | 高 | 追问"为什么需要这么说" |
| 享乐（jouissance） | 高 | 识别痛苦中的快感 |
| 幻想 | 高 | 识别维持矛盾的叙事框架 |
| 矛盾 | 高 | 找到具体的张力 |
| 症状 | 中 | 识别"问题"的隐藏功能 |
| 大他者 | 中 | 追问隐含的审判目光 |
| 实在界 | 中 | 识别叙事的断裂点 |
| 残缺 | 中 | 理解欲望的结构性起点 |
| 主体 | 低 | 追问话语征召的位置 |
| 穿越幻想 | 低 | 分析的终极动作 |
| 症候 | 极低 | 与模式认同而非克服 |
